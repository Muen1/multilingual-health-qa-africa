# Multilingual Health Question Answering in Low-Resource African Languages

## Overview
This is a fine-tuning pipeline for multilingual health QA across low-resource African languages, built for the Zindi [Multilingual Health QA Challenge](https://zindi.africa/). The project systematically fine-tunes sequence-to-sequence models (mt5-small, flan-t5-base, flan-t5-large) using LoRA/PEFT, progressing through 6 experiments evaluated by ROUGE-1, ROUGE-L, and LLM-as-a-Judge.

## Data 

The dataset is from the Zindi Multilingual Health QA Challenge. It contains health-related questions and answers across multiple low-resource African languages (subset column). Key statistics:

* Train: 29,814 samples (after removing 1 empty row)

* Validation: 6,686 samples

* Test: 2,618 samples (no ground-truth labels)

* Languages: Multiple African languages identified by the subset column (e.g., Aka_Gha)


The 02_preprocessing.ipynb notebook performs Unicode normalization (NFC), control character removal, whitespace normalization, and filters answers exceeding 512 words. Cleaned files are saved as `train_clean.csv`, `val_clean.csv`, and `test_clean.csv`.

## Evaluation Metrics

The Zindi leaderboard uses a composite score averaging:

* ROUGE-1 F1 — unigram overlap between generated and reference answers

* ROUGE-L F1 — longest common subsequence-based overlap

* LLM-as-a-Judge — semantic quality scored by a large language model


Local validation uses ROUGE-1 and ROUGE-L via the HuggingFace `evaluate` library. Per-language ROUGE breakdown is computed in `04_results_summary_fixed.ipynb`.

## Reproducing a Submission

After training any experiment, the submission file is generated automatically:

```bash
# Inside each fine-tuning notebook — final cells
answers = generate_answers(model, tokenizer, test['input_text'].tolist())
save_predictions(test, answers, experiment_name)
build_submission(experiment_name)
# → saves to: multilingual_health_qa/submissions/submission_{experiment_name}.csv
```

Upload the generated CSV to Zindi.
